String Permutations

Backtracking Medium

Description

The String Permutations algorithm generates all possible arrangements of characters in a given string. For a string of length n, there are n! (n factorial) possible permutations. This algorithm uses backtracking to systematically generate all these combinations by building the permutations one character at a time.

History

The concept of permutations dates back to ancient civilizations. The systematic study of permutations began in the 12th century with Indian mathematicians. The modern algorithmic approach to generating permutations was developed in the early days of computer science. Notable contributions include Steinhaus-Johnson-Trotter algorithm (1960s) and Heap's algorithm (1963). The backtracking approach we use today became popular due to its simplicity and adaptability to various programming problems.

def generate_permutations(s: str) -> list[str]:
    def backtrack(pos: int):
        if pos == len(s):
            result.append(''.join(current))
            return
            
        for i in range(len(s)):
            if not used[i]:
                used[i] = True
                current[pos] = s[i]
                backtrack(pos + 1)
                used[i] = False
    
    result = []
    current = [''] * len(s)
    used = [False] * len(s)
    backtrack(0)
    return result

class StringPermutations {
private:
    void backtrack(string& s, vector<bool>& used, string& current, 
                  vector<string>& result, int pos) {
        if (pos == s.length()) {
            result.push_back(current);
            return;
        }
        
        for (int i = 0; i < s.length(); i++) {
            if (!used[i]) {
                used[i] = true;
                current[pos] = s[i];
                backtrack(s, used, current, result, pos + 1);
                used[i] = false;
            }
        }
    }
    
public:
    vector<string> generatePermutations(string s) {
        vector<string> result;
        vector<bool> used(s.length(), false);
        string current(s.length(), ' ');
        backtrack(s, used, current, result, 0);
        return result;
    }
};

int main() {
    string s = "ABC";
    StringPermutations solution;
    vector<string> permutations = solution.generatePermutations(s);
    
    cout << "All permutations of " << s << ":\n";
    for (const string& perm : permutations) {
        cout << perm << endl;
    }
    
    return 0;
}
                                        

public class StringPermutations {
    private void Backtrack(string s, bool[] used, char[] current, 
                         List<string> result, int pos) {
        if (pos == s.Length) {
            result.Add(new string(current));
            return;
        }
        
        for (int i = 0; i < s.Length; i++) {
            if (!used[i]) {
                used[i] = true;
                current[pos] = s[i];
                Backtrack(s, used, current, result, pos + 1);
                used[i] = false;
            }
        }
    }
    
    public List<string> GeneratePermutations(string s) {
        var result = new List<string>();
        var used = new bool[s.Length];
        var current = new char[s.Length];
        Backtrack(s, used, current, result, 0);
        return result;
    }
}

public class Program {
    public static void Main() {
        string s = "ABC";
        var solution = new StringPermutations();
        var permutations = solution.GeneratePermutations(s);
        
        Console.WriteLine($"All permutations of {s}:");
        foreach (var perm in permutations) {
            Console.WriteLine(perm);
        }
    }
}

Complexity Analysis

Time Complexity

The time complexity is O(n!), where n is the length of the string:

  • For first position: n choices
  • For second position: (n-1) choices
  • For third position: (n-2) choices
  • And so on...
  • Total: n * (n-1) * (n-2) * ... * 1 = n!

Space Complexity

The space complexity is O(n), which includes:

  • Recursion stack space: O(n)
  • Used array: O(n)
  • Current permutation array: O(n)
  • Note: Result storage is O(n * n!), but this is typically not counted

Advantages and Disadvantages

Advantages

  • Simple and intuitive implementation
  • Generates all possible permutations
  • Works well for small strings
  • Can be modified to handle duplicates

Disadvantages

  • Factorial time complexity
  • Not suitable for long strings
  • Memory usage grows with string length
  • May be overkill when only some permutations are needed
